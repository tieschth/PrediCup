# Развёртывание PrediCup через Docker

Инструкция для того, кто разворачивает бота на сервере/ПК. Код бота уже готов —
нужно только заполнить два файла с настройками и запустить контейнер.

## 0. Что потребуется заранее

На машине должны быть установлены:
- **Docker** и **Docker Compose** (входит в Docker Desktop / `docker compose`);
- **git** (или просто скачать архив репозитория).

И значения, которые даст владелец бота (их нет в репозитории — заполняются вручную):
- `BOT_TOKEN` — токен Telegram-бота;
- `FOOTBALL_API_KEY` — ключ football-data.org;
- список `tg_id` администраторов;
- `id` чата (группы), где работает бот (вид `-100…`).

## 1. Получить код

```bash
git clone https://github.com/tieschth/PrediCup.git
cd PrediCup
```

## 2. Создать файл секретов `.env`

Файла `.env` в репозитории нет (секреты не хранятся в git) — его нужно создать
вручную. Скопируй шаблон и заполни:

```bash
cp .env.example .env
```

Открой `.env` и впиши значения:
```
BOT_TOKEN=сюда_токен_бота
FOOTBALL_API_KEY=сюда_ключ_football_data_org
ENV=prod
```
> `ENV=prod` обязательно для боевого режима — это отключает тестовые команды
> (`/devmatch`, `/devresult` и т.п.). Пути `CONFIG_PATH`/`DB_PATH` менять не нужно —
> внутри контейнера они уже заданы через docker-compose.

## 3. Создать конфиг `config/config.yaml`

Этого файла тоже нет в репозитории. Скопируй пример и заполни:

```bash
cp config/config.example.yaml config/config.yaml
```

В `config/config.yaml` укажи как минимум:
```yaml
bot:
  vote_open_hours_before: 10        # за сколько часов до матча открывать голосование
  display_timezone: "Europe/Moscow" # часовой пояс отображения времени матчей

provider:
  name: football_data_org
  competition: "WC"

roles:
  admins: [123456789]               # tg_id админов
  allowed_chats: [-1001234567890]   # id чата (группы)
```
Остальные поля можно оставить по умолчанию.

## 4. Запустить бота в фоне

```bash
docker compose up -d --build
```
- `-d` — запуск в фоне (бот продолжит работать после закрытия терминала);
- `--build` — собрать образ (нужно при первом запуске и после обновления кода).

Проверить, что бот поднялся:
```bash
docker compose logs -f
```
В логах должно появиться: `Стартовая синхронизация: 104 матчей` и
`Бот запущен. Провайдер: football_data_org`. Выйти из просмотра логов — `Ctrl+C`
(контейнер при этом продолжает работать).

## 5. Управление

```bash
docker compose ps          # статус контейнера
docker compose logs -f     # смотреть логи в реальном времени
docker compose restart     # перезапустить
docker compose down        # остановить и удалить контейнер (данные сохранятся)
docker compose up -d        # снова запустить
```

## 6. Данные и перезапуск

База SQLite лежит в папке `./data` на хосте (том примонтирован в контейнер),
поэтому **прогнозы и очки переживают перезапуск/пересборку контейнера**. Удалять
`./data` не нужно. Резервная копия — просто скопировать файл `./data/predicup.sqlite3`.

## 7. Обновление кода

```bash
git pull
docker compose up -d --build
```
`.env`, `config/config.yaml` и `./data` при этом не затрагиваются.

## 8. Важно: машина не должна «засыпать»

Бот работает, пока запущен контейнер И включён хост. Если компьютер уходит в
**сон/гибернацию**, процесс приостанавливается: бот не принимает голоса и не
подводит итоги, пока машина не проснётся. Для стабильной работы отключи на хосте
переход в сон (или используй всегда включённый сервер). Сам Docker сон хоста не
предотвращает.

> Пропущенное во время сна не теряется безвозвратно: Telegram удерживает
> непрочитанные апдейты ~24 часа, а итоги матчей бот подведёт на ближайшем цикле
> проверки после пробуждения. Но рассчитывать на это как на штатный режим не стоит.
